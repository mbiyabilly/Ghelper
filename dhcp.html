Installation et Configuration d'un Serveur DHCP

Un serveur DHCP (Dynamic Host Configuration Protocol) est essentiel dans les réseaux professionnels pour attribuer automatiquement des adresses IP, DNS et passerelles aux appareils connectés.

Ce guide explique comment :

  • Installer et configurer un serveur DHCP sous Linux
  • Gérer les plages d'adresses IP dynamiques et statiques
  • Configurer le NAT pour permettre l'accès Internet
  • Surveiller et dépanner le service DHCP

Schéma Réseau

+-------------------+ +-------------------+ +----------------------+ | Client DHCP | | Serveur DHCP | | Routeur/Passerelle | | (Obtenir IP auto) | <---> | (Linux - enp0s3) | <---> | (192.168.10.254) | +-------------------+ +-------------------+ +----------------------+ ▲ | | ▼ +-------------------+ +-------------------+ | Internet | | Autres Clients | | (via NAT) | | (DHCP ou Static) | +-------------------+ +-------------------+

Installation des paquets



sudo apt update
sudo apt install isc-dhcp-server iptables-persistent vim -y

Configuration IP statique



sudo vi /etc/netplan/01-static-ip.yaml




network:
 version: 2
 renderer: networkd
 ethernets:
   enp0s3:
     addresses:
       - 192.168.10.2/24
     nameservers:
       addresses: [8.8.8.8,8.8.4.4]
     routes:
       - to: default
         via: 192.168.10.254




sudo netplan apply

Définir l'interface DHCP



sudo cp /etc/default/isc-dhcp-server /etc/default/isc-dhcp-server.bkp
echo 'INTERFACESv4="enp0s3"' | sudo tee /etc/default/isc-dhcp-server

Configuration DHCP



sudo cp /etc/dhcp/dhcpd.conf /etc/dhcp/dhcpd.conf.bkp
sudo vim /etc/dhcp/dhcpd.conf




option domain-name "example.local";
option domain-name-servers 8.8.8.8,1.1.1.1;
default-lease-time 600;
max-lease-time 7200;
ddns-update-style none;
authoritative;

subnet 192.168.10.0 netmask 255.255.255.0 {
 range 192.168.10.50 192.168.10.200;
 option routers 192.168.10.254;
 option subnet-mask 255.255.255.0;
}

host reserved-client {
 hardware ethernet 08:00:27:bd:22:0c;
 fixed-address 192.168.10.50;
}

Démarrage du service



sudo dhcpd -t
sudo systemctl restart isc-dhcp-server
sudo systemctl enable isc-dhcp-server
sudo systemctl status isc-dhcp-server

Activation du NAT



echo "net.ipv4.ip_forward=1" | sudo tee -a /etc/sysctl.conf
sudo sysctl -p




sudo iptables -t nat -A POSTROUTING -o enp0s8 -j MASQUERADE
sudo iptables -A FORWARD -i enp0s3 -o enp0s8 -m state --state RELATED,ESTABLISHED -j ACCEPT
sudo iptables -A FORWARD -i enp0s8 -o enp0s3 -j ACCEPT
sudo iptables-save | sudo tee /etc/iptables/rules.v4

Surveillance et vérification



sudo dhcp-lease-list
tail -f /var/log/syslog | grep dhcpd
journalctl -xeu isc-dhcp-server.service

Dépannage côté client



sudo dhclient -v
ip a
