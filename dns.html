Installation et Configuration d'un Serveur DNS

Un serveur DNS (Domain Name System) est indispensable dans les infrastructures réseau pour résoudre les noms de domaine en adresses IP, permettant aux machines de communiquer par nom plutôt que par adresse numérique.

Ce guide explique comment :

  • Installer et configurer BIND9 sous Ubuntu
  • Gérer les zones DNS directes et inverses
  • Configurer le pare-feu et forcer l'IPv4
  • Tester et sécuriser le service DNS

Schéma Réseau

+-------------------+ +-------------------+ +----------------------+ | Client DNS | | Serveur DNS | | Forwarder/Internet | | (Résolution noms) | <---> | (BIND9 - enp0s8) | <---> | (8.8.8.8 / 1.1.1.1)| +-------------------+ +-------------------+ +----------------------+ ▲ | | | ▼ ▼ +-------------------+ +--------------------+ +-------------------+ | diarabaka.local | | Zone Inverse | | Internet | | (DNS Forward) | | 129.168.192.arpa | | (DNS Externe) | +-------------------+ +--------------------+ +-------------------+

Step 1 Configuration IP statique (Netplan)

Avant d'installer BIND9, configurez une adresse IP statique sur l'interface réseau afin que le serveur DNS soit toujours joignable à la même adresse.


sudo nano /etc/netplan/00-installer-config.yaml
  

network:
  version: 2
  ethernets:
    enp0s3:
      dhcp4: false
    enp0s8:
      addresses: [192.168.129.172/24]
      nameservers:
        addresses: [1.1.1.1, 8.8.8.8]
  

sudo netplan apply

Step 2 Installation de BIND9

Installez BIND9 ainsi que ses utilitaires via apt. Le paquet bind9-utils fournit des outils de diagnostic comme named-checkconf et named-checkzone.


sudo apt update
sudo apt install bind9 bind9-utils -y

Step 3 Activation du Pare-feu

Activez UFW et autorisez BIND9 à recevoir des requêtes DNS sur le port 53 (TCP et UDP).


sudo ufw enable
sudo ufw allow bind9
sudo ufw status

Step 4 Démarrage du service BIND9

Démarrez le service et vérifiez qu'il est bien actif avant de procéder à la configuration.


sudo systemctl start bind9
sudo systemctl status bind9

Step 5 Configuration principale du DNS

Éditez le fichier named.conf.options pour définir les options globales : écoute sur toutes les interfaces, restriction des requêtes au réseau local, et forwarder vers Google DNS.


sudo nano /etc/bind/named.conf.options
  

options {
    directory "/var/cache/bind";
    listen-on { any; };
    allow-query { localhost; 192.168.129.0/24; };

    forwarders {
        8.8.8.8;
    };

    dnssec-validation no;
};

Step 6 Forcer l'IPv4

Par défaut, BIND9 peut tenter d'utiliser IPv6. Forcez l'IPv4 en modifiant les options de démarrage du daemon, puis vérifiez la configuration et redémarrez.


sudo nano /etc/default/named
  

OPTIONS="-u bind -4"
  

sudo named-checkconf
sudo systemctl restart bind9

Step 7 Déclaration des Zones DNS

Déclarez la zone directe diarabaka.local et la zone inverse dans named.conf.local. Le type master indique que ce serveur fait autorité sur ces zones.


sudo nano /etc/bind/named.conf.local
  

zone "diarabaka.local" IN {
    type master;
    file "/etc/bind/zones/db.diarabaka.local";
};

zone "129.168.192.in-addr.arpa" {
    type master;
    file "/etc/bind/zones/db.129.168.192";
};

Step 8 Création des fichiers de zone

Créez le répertoire des zones et copiez le fichier modèle db.local comme base pour votre zone DNS locale.


sudo mkdir -p /etc/bind/zones
sudo cp /etc/bind/db.local /etc/bind/zones/db.diarabaka.local

Step 9 Tests DNS

Vérifiez que la résolution de noms fonctionne correctement — dans les deux sens (nom → IP et IP → nom).


host dns01.diarabaka.local
ping dns01.diarabaka.local
host dns01
host 192.168.129.172

Step 10 Sécurité

Assurez-vous que les fichiers de zone appartiennent bien à l'utilisateur bind afin d'éviter tout problème de permissions au démarrage du service.


sudo chown -R bind:bind /etc/bind/zones
